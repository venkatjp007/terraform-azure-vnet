# VNET Module

Reusable Terraform module that provisions an Azure Virtual Network with a
configurable set of subnets and optional per-subnet Network Security Groups.

## Features

- Virtual Network with configurable address space and optional custom DNS.
- Arbitrary number of subnets driven by a single `subnets` map variable.
- Optional NSG creation and rule assignment per subnet (`create_nsg`, `nsg_rules`).
- Optional service endpoints per subnet.
- Consistent tagging on every created resource.
- Input validation on names and CIDR blocks.

## Usage

```hcl
module "vnet" {
  source              = "../../modules/vnet"
  name                = "opella-dev-eus-vnet"
  resource_group_name = azurerm_resource_group.this.name
  location            = "eastus"
  address_space       = ["10.10.0.0/16"]

  subnets = {
    app = {
      address_prefixes = ["10.10.1.0/24"]
      create_nsg       = true
      nsg_rules = [{
        name                   = "allow-ssh-from-vnet"
        priority               = 100
        direction              = "Inbound"
        access                 = "Allow"
        protocol               = "Tcp"
        destination_port_range = "22"
        source_address_prefix  = "VirtualNetwork"
      }]
    }
    data = {
      address_prefixes  = ["10.10.2.0/24"]
      service_endpoints = ["Microsoft.Storage"]
    }
  }

  tags = {
    environment = "dev"
    managed_by  = "terraform"
  }
}
```

## Testing

Plan-time tests live in `tests/vnet.tftest.hcl`:

```bash
cd modules/vnet
terraform init -backend=false
terraform test
```
